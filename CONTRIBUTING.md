# Contributing to Zyrcon

Thanks for taking the time to contribute.

Zyrcon is being built as a practical system for trustworthy AI operators. We care about ideas, but we care even more about changes that improve reliability, clarity, and real-world usefulness.

## Before you contribute

Please read:
- [README](./README.md)
- [Code of Conduct](./CODE_OF_CONDUCT.md)
- [Security Policy](./SECURITY.md)

## Best ways to contribute

- report reproducible bugs
- improve setup or documentation
- suggest focused workflow improvements
- contribute examples or starter operators
- improve test coverage and reliability

## What good contributions look like

A strong contribution is:
- clear
- scoped
- testable
- respectful of existing architecture
- focused on real operator usefulness

## Workflow

1. Fork the repository
2. Create a branch with a clear name
3. Make one focused change
4. Add or update tests if applicable
5. Open a pull request with context

## Pull request expectations

Please include:
- what changed
- why it changed
- how it was tested
- screenshots or logs if relevant
- any follow-up work still needed

## Style guidelines

- prefer clarity over cleverness
- document non-obvious decisions
- keep names literal and understandable
- avoid unnecessary abstractions
- optimize for maintainability

## If you are unsure

Open a Discussion first for:
- larger feature ideas
- architecture changes
- unclear product direction questions

Open an Issue first for:
- bug reports
- focused feature requests
- concrete tasks

Thanks for helping build something useful.

# Security Policy

## Reporting a vulnerability

Please do **not** report security vulnerabilities in public issues or discussions.

Instead, email:

**security@zyrconlabs.com**

Include:
- a clear description of the issue
- steps to reproduce
- affected files or components
- impact assessment if known
- screenshots, logs, or proof of concept if helpful

## What to expect

We will try to:
- acknowledge receipt
- assess severity
- determine mitigation or fix path
- communicate next steps when possible

## Scope

Security issues may include:
- approval bypass
- credential exposure
- unsafe execution paths
- privilege escalation
- insecure defaults
- unintended data leakage

## Good-faith testing

We welcome responsible disclosure done in good faith and with minimal impact.

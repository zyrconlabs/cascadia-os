# Production Deployment Checklist ŌĆö Cascadia OS

Use this checklist before serving live traffic. Each section must be fully complete before moving to the next.

---

## 1. Pre-Deployment ŌĆö Server & Environment

- [ ] Server meets minimum requirements: Python 3.11+, 4 GB RAM, 20 GB storage
- [ ] Operating system is Ubuntu 22.04 LTS or macOS 13+ (Ventura or later)
- [ ] Firewall configured: port 6300 (PRISM) accessible from intended clients only; all other Cascadia ports (5100ŌĆō6205) bound to `127.0.0.1`
- [ ] HTTPS reverse proxy configured (nginx or Caddy) in front of PRISM on port 6300 if exposing externally
- [ ] SSH access secured: key-based auth only, password auth disabled
- [ ] Non-root user created for running Cascadia OS process (`adduser cascadia`)
- [ ] `ulimit -n` Ōēź 4096 (required for concurrent operator connections)
- [ ] NTP synchronized: `timedatectl status` shows `NTP service: active`
- [ ] Backup storage mounted and writable at `data/backups/`

---

## 2. Configuration Hardening

Run `GET /api/prism/production` after startup ŌĆö all items must show `production_ready: true`.

- [ ] `config.json` created from `config.example.json` ŌĆö **not** using example file directly
- [ ] `curtain.signing_secret` replaced with output of:
  ```
  python3 -c "import secrets; print(secrets.token_hex(32))"
  ```
- [ ] `license_secret` replaced with a separate random secret (use same command above)
- [ ] `license_key` populated with a valid license key generated by `scripts/generate_license.py`
- [ ] `sentinel_fail_open` set to `false` (default ŌĆö do **not** set to `true` in production)
- [ ] `stripe.webhook_secret` set from Stripe Dashboard ŌåÆ Developers ŌåÆ Webhooks ŌåÆ Signing Secret
- [ ] `approval_escalation_email` set to a monitored mailbox
- [ ] `weekly_summary_email` set (or `reports_dir` confirmed writable for local report delivery)
- [ ] `operators_dir` set to correct path if using commercial operators (not the bundled default)
- [ ] LLM configured: `llm.provider`, `llm.model`, and `llm.base_url` set; `llm.configured: true`
- [ ] `data/` directory structure initialized: `data/runtime/`, `data/logs/`, `data/backups/`

---

## 3. Service Startup & Health

- [ ] Run `python3 watchdog.py --config config.json` ŌĆö no CONFIG warnings logged
- [ ] All Tier 1 components reach `ready` state: `crew`, `vault`, `sentinel`, `curtain`
- [ ] All Tier 2 components reach `ready` state: `beacon`, `stitch`, `vanguard`, `handshake`, `bell`, `almanac`
- [ ] PRISM (Tier 3) reaches `ready` state
- [ ] PRISM health check passes: `curl http://127.0.0.1:6300/api/prism/health-check`
- [ ] Production readiness endpoint confirms all clear: `curl http://127.0.0.1:6300/api/prism/production`
  - Expected: `{ "production_ready": true, "issues": [] }`
- [ ] No `ERROR` or `WARNING` lines in `data/logs/` within first 60 seconds of startup (CONFIG warnings from above must be resolved before this check)
- [ ] Operator autostart confirmed: all operators in registry show `status: ok` in PRISM ŌåÆ Operators tab
- [ ] Approval timeout daemon confirmed running (check `data/logs/prism.log` for "approval timeout daemon started")

---

## 4. Payments & Licensing

- [ ] WooCommerce store live at `zyrcon.store` ŌĆö test purchase completes end-to-end
- [ ] Stripe webhook endpoint registered and receiving events (check Stripe Dashboard ŌåÆ Events)
- [ ] Stripe webhook secret matches `stripe.webhook_secret` in `config.json`
- [ ] Test license key delivery: place a $0 test order in Stripe test mode; verify key arrives in email
- [ ] Serial number inventory stocked: at least 100 keys per tier in WooCommerce Serial Numbers
  ```
  python3 scripts/generate_bulk_licenses.py --tier pro --count 500 --days 365
  python3 scripts/generate_bulk_licenses.py --tier business-starter --count 200 --days 365
  ```
- [ ] Apple Pay enabled and domain verified (Safari shows Apple Pay button at checkout)
- [ ] Payment links configured in `config.enterprise.json` ŌåÆ `payment_links` (for upgrade prompt)
- [ ] License activation flow tested: enter key in Cascadia OS setup ŌåÆ activates without error

---

## 5. Monitoring & Alerts

- [ ] Log rotation configured: `data/logs/` rotated weekly, 4 weeks retention minimum
- [ ] Systemd service installed and enabled (`systemctl status cascadia` ŌåÆ active)
  ```
  systemctl enable cascadia
  systemctl start cascadia
  ```
- [ ] Watchdog restart confirmed: stop FLINT manually, verify watchdog restarts it within 30 seconds
- [ ] Approval escalation email tested: create a HIGH-risk approval, let it sit past 30 minutes; verify escalation email received at `approval_escalation_email`
- [ ] Weekly summary delivery tested: trigger a manual summary or verify `reports_dir` contains output
- [ ] Server health check script passing:
  ```
  bash hardware/server_health_check.sh
  ```
  Expected: all checks `OK`, no `FAIL` lines
- [ ] PRISM system monitor returning live data: `curl http://127.0.0.1:6300/api/prism/system/monitor`
- [ ] Off-site backup configured and first backup confirmed in `data/backups/`
- [ ] Incident runbook location documented for on-call team

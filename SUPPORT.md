# Support

## Where to get help

### GitHub Discussions
Use Discussions for:
- setup questions
- product feedback
- roadmap input
- showing what you built

### GitHub Issues
Use Issues for:
- reproducible bugs
- focused feature requests
- documentation problems
- specific tasks

## Before opening an issue

Please include:
- what you expected
- what actually happened
- environment details
- logs, screenshots, or steps to reproduce

## Security problems

Do **not** open public issues for sensitive security problems.

Use the instructions in [SECURITY.md](./SECURITY.md).

## Launch note

If something in the early public release is unclear, open a Discussion. Early confusion is useful feedback.
